# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly.

### How to Report

1. **Email**: Send details to flongo11@gmail.com
2. **Subject Line**: "Security Vulnerability Report - Criminal Investigation Request"
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: Within 48 hours of your report
- **Assessment**: We will investigate and assess the severity
- **Resolution**: Security fixes will be prioritized and deployed as soon as possible
- **Credit**: You will be credited for responsible disclosure (if desired)

### Security Considerations

This repository contains:
- Static HTML documentation
- Client-side JavaScript for evidence failover
- LocalStorage for access logging

### Out of Scope

- Issues related to external evidence hosting services
- General web hosting security (GitHub Pages)
- Third-party browser vulnerabilities

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

## Security Best Practices

When using or hosting this documentation:

1. Serve over HTTPS only
2. Implement Content Security Policy headers
3. Regular security audits of embedded JavaScript
4. Monitor evidence link availability
5. Keep backup copies of evidence materials

---

**Note**: This is a documentation repository for legal/investigative purposes. Security issues should be reported promptly to maintain evidence integrity.
