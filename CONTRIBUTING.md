# Contributing Guidelines

Thank you for your interest in this project. Due to the sensitive legal nature of this criminal investigation documentation, contributions are managed carefully.

## Important Context

This repository contains formal legal documentation for an active criminal investigation request. All content is based on verified evidence and legal submissions.

## Types of Contributions

### Accepted Contributions

1. **Technical Improvements**
   - Bug fixes in HTML/CSS/JavaScript
   - Accessibility enhancements
   - Performance optimizations
   - Cross-browser compatibility fixes
   - Security improvements

2. **Documentation**
   - Typo corrections
   - Clarity improvements
   - Technical documentation updates

3. **Infrastructure**
   - CI/CD improvements
   - Deployment automation
   - Testing infrastructure

### Not Accepted

- Changes to factual content or evidence claims
- Modifications to legal arguments
- Removal or alteration of documented evidence
- Subjective editorial changes to case details

## How to Contribute

### For Technical Issues

1. **Open an Issue First**
   - Describe the technical problem
   - Provide steps to reproduce
   - Suggest a solution if possible

2. **Submit a Pull Request**
   - Fork the repository
   - Create a feature branch (`git checkout -b fix/issue-description`)
   - Make your changes
   - Test thoroughly
   - Commit with clear messages
   - Submit PR with detailed description

### Code Standards

- **HTML**: Valid HTML5, semantic markup
- **CSS**: Organized, commented, follows existing patterns
- **JavaScript**: Clear, documented, minimal dependencies
- **Accessibility**: WCAG 2.1 AA compliance
- **Performance**: Maintain fast load times

### Testing

Before submitting:
- [ ] HTML validates (W3C validator)
- [ ] CSS validates
- [ ] JavaScript has no console errors
- [ ] Works offline
- [ ] Prints correctly
- [ ] Mobile responsive
- [ ] Cross-browser tested (Chrome, Firefox, Safari, Edge)

## Review Process

1. All PRs are reviewed by the repository maintainer
2. Technical changes are evaluated for impact
3. Legal content changes require additional verification
4. Approved PRs are merged and deployed
